Description of Application

This web application is a Synchronous Chat Service. The first tier of the application is the client's interface to the chatroom. This will be implemented using Java Server Faces to allow dynamic creation of html at runtime. To stop the entire page needing to be refreshed every time there is an update, ajax will be used to allow partial page updates, providing a more friendly user experience. This will run on a Java web server, most likely to be tomcat. The middle or second tier of the application will also run on the web server an act as the communication layer between the first and third tiers. This tier will function as a servlet and will alter the requests sent to it by the first tier and pass them onto the third tier. This tier acts like a XMPP client. The third tier is essentially the XMPP server, which will relay chat messages back through to the tiers to the client.

Descriptions of pages

Portal Web Chat Client
This is the page that the user arrives at when first accessing the application. The user can either enter the application as a registered account user by entering in a valid username and password to the form; or as a guest, by clicking the tickbox "enter as a guest". The user can decide which room they wish to enter at this stage, and they will be sent direct to that page, bypassing the main chatroom page. On submitting the information above, on clicking the "enter" button they will be directed to the chatrooms.

If a user wishes to create an account they can access the account creation page from here, by use of the "create account" button. The report page is also accessible from this page by the use of the "about project" button.

Chat Rooms
The chat room page will be the main page of the application. It will contain a chat box to show received and sent messages, a send message input box and a side panel. The top of the chat box will contain tabs to show the different rooms the user is currently in. Each tab will contain the name of a room and by clicking on it the user will be able change rooms.

The side panel will consist of three tabs, friends, rooms and users. We plan to develop the friends feature if we have time. This will show all the friends that the current user has. The rooms tab will show all the rooms currently created. The user can the use this to quickly join other rooms. This tab will also show how many people are currently in each room. Account Creation
The account page allows the user to create an account for accessing the chat rooms. This will enhance the users experience by offering them the ability to retain their identity across many sessions. The page is set up to operate on an HTML5 form. The user enters their requested username, on moving away from that field the server will check whether that username is available. If not the user will be informed, and offered the chance to create a different username. The user then enters a password into an HTML5 password type text entry. They then are required to re-enter the password, this will be checked to see it matches, on the user moving off that element. They then enter their email address, this must be of the type set by the HTML5 email type. The user then re-enters this, and again the server checks whether the two entries match when the user moves off that element. The user then enters their date of birth, into an HTML5 date type input field. They can describe themselves in one sentence if they wish.

Currently the information in the profile page is for account creation, in order to allow the user to choose a username. However, we have a potential plan to develop a profile page that other users can click on to see information about a given user. If we do include this option, the tick box contained within the account creation page, (currently obsolete) would then allow a user to enter details about themselves, but choose to have them hidden from other users if they wish.

There are also 3 buttons on this page. "about project" brings the user to this report page. "register" sends the details to be attached to the database. "cancel" takes the user back to the main entrance page, from where they can enter the chat room as a guest if they wish.
